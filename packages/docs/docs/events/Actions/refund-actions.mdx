# Refund Actions Reference

### checkout_engine/refund_created

Occurs when a new refund is created. This does not guarantee that the refund will be successful.

```php
do_action(
	'checkout_engine/refund_created',
	(string) $refund_id,
	(array) $webhook_data
)
```

### checkout_engine/refund_updated

Occurs when a refund is updated.

```php
do_action(
	'checkout_engine/refund_updated',
	(string) $refund_id,
	(array) $webhook_data
)
```

### checkout_engine/refund_succeeded

Occurs when a refund has succesfully been processed. This is typically when you'll want to revoke access to something.

```php
do_action(
	'checkout_engine/refund_succeeded',
	(string) $refund_id,
	(array) $webhook_data
)
```

### checkout_engine/refund_failed

Occurs when a refund has failed.

```php
do_action(
	'checkout_engine/refund_failed',
	(string) $refund_id,
	(array) $webhook_data
)
```

### checkout_engine/refund_canceled

Occurs when a refund has been canceled

```php
do_action(
	'checkout_engine/refund_canceled',
	(string) $refund_id,
	(array) $webhook_data
)
```
