# Subscription Actions Reference

### checkout_engine/subscription_created

Occurs when a new subscription is created. This is typically in `incomplete` status but it could be in `trialing`, `active`, `past_due`, or `canceled`.

```php
do_action(
	'checkout_engine/subscription_created',
	(string) $subscription_id,
	(array) $webhook_data
)
```


### checkout_engine/subscription_canceled

Occurs whenever a subscription is canceled. Typically this is when you'll revoke access to something. Note that this only occurs after
a customers subscription period ends. If their subscription is pending cancellation, it will not trigger this event.

```php
do_action(
	'checkout_engine/subscription_canceled',
	(string) $subscription_id,
	(array) $webhook_data
)
```

### checkout_engine/subscription_made_active

Occurs whenever a subscription is made active. Typically this is when you'll give access to something. Note that this can occur if a user
is already in the trialing state, too.

```php
do_action(
	'checkout_engine/subscription_made_active',
	(string) $subscription_id,
	(array) $webhook_data
)
```

### checkout_engine/subscription_made_trialing

Occurs when a new subscriptions transitions into a trial state. If the subscription has a trial, it will start with this status until the end of the trial period.

```php
do_action(
	'checkout_engine/subscription_made_trialing',
	(string) $subscription_id,
	(array) $webhook_data
)
```


### checkout_engine/subscription_updated

Occurs when a subscription is updated. This can happen if any part of the subscription updates (items, periods, status, etc.).

```php
do_action(
	'checkout_engine/subscription_updated',
	(string) $subscription_id,
	(array) $webhook_data
)
```
